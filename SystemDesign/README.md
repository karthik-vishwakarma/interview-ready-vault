# System Design Interview Vault

System Design interviews test your ability to design large-scale distributed systems. This section focuses on
architectural patterns, trade-offs (CAP Theorem), and real-world case studies.

## 🏗️ Core Pillars

Each sub-topic contains concepts and design patterns essential for high-level architecture.

### 1. [Foundational Concepts](./Concepts/README.md)

* **Topics:** Load Balancing, Caching, Sharding, Proxies, and Content Delivery Networks (CDNs).
* **Focus:** The building blocks of any scalable system.

### 2. [Databases & Storage](./Databases/README.md)

* **Topics:** SQL vs. NoSQL, Replication, Consistent Hashing, and Indexing.
* **Focus:** Data consistency, availability, and partitioning.

### 3. [Communication Patterns](./Communication/README.md)

* **Topics:** REST vs. gRPC, WebSockets, Message Queues (Kafka/RabbitMQ), and Pub/Sub.
* **Focus:** How services talk to each other efficiently.

### 4. [Real-World Design Problems](./CaseStudies/README.md)

* **Topics:** Design a URL Shortener, Twitter Feed, WhatsApp, or Rate Limiter.
* **Focus:** End-to-end architecture and component interaction.
