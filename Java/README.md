# Java Interview Topics

Select a sub-topic to begin practicing. Each section contains questions classified by difficulty.

## 🔍 Sub-Topics

1. [Core & Language Fundamentals](Core&LanguageFundamentals/README.md)
    * Concepts: Data types, Autoboxing, OOPs (Inheritance, Polymorphism, etc.), Exception Handling, and Strings.
    * Why: The foundation of every technical screening.
2. Collections Framework
    * Concepts: List, Set, Map, Queue, Fail-fast vs Fail-safe, and Internal Hashing mechanics.
    * Why: Crucial for data manipulation and algorithmic efficiency questions.
3. [Multithreading & Concurrency](Multithreading&Concurrency/README.md)
    * Concepts: Thread Lifecycle, synchronized, volatile, Executors, ReentrantLocks, and CompletableFuture.
    * Why: Tests your ability to write scalable, thread-safe applications.
4. JVM Internals & Garbage Collection
    * Concepts: ClassLoaders, JVM Memory Model (Heap/Stack/Metaspace), GC Algorithms (G1, ZGC), and JIT Compilation.
    * Why: Important for performance tuning and memory leak troubleshooting.
5. Java 8+ Features & Functional Programming
    * Concepts: Lambdas, Stream API, Optional, Default Methods, and Var-type.
    * Why: Modern Java development relies heavily on functional paradigms.
6. Advanced Language Features
    * Concepts: Reflection API, Annotations, Generics (Wildcards/Erasure), and Serialization.
    * Why: Usually asked in Senior or Architect level interviews.
7. Input/Output & Networking
    * Concepts: IO vs NIO, Buffering, Socket Programming, and Serialization protocols.
    * Why: Relevant for backend systems involving file processing or low-level communication.
8. Modern Java (11, 17, 21)
    * Concepts: Records, Sealed Classes, Pattern Matching, and Virtual Threads (Project Loom).
    * Why: Demonstrates that you stay updated with the latest LTS versions.
